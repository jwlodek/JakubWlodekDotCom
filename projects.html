Focus Stacked 3D Reconstruction

During my internship at Brookhaven National Laboratory, I constructed software that used focus stacking as a way of collecting high quality images with a shallow depth of field fixed microscope, as well as recreating three dimensional depth data. The video to the left shows some of my results. It is the result of collecting 18 images of a mineral with a 5x microscope at a shallow depth of focus. The images were then merged into one, to create a single in-focus image. Finally, using OpenGL, the merged image, along with depth information that was collected simultaneously was used to generate the 3D model. This project has resulted in two co-authored publications, one of which, "On-Axis 3D Microscope For X-Ray Beamlines at NSLS-II," is linked above. The second half of the publication discusses the method used in writing the software, and shows some more sample images. Additionally, another 3D video has been uploaded to the youtube channel linked above.


ADPluginBar EPICS Barcode reader plugin

As a way to simplify sample managment at National Synchrotron Light Source II beamlines, I worked on constructing a barcode reader plugin for the area detector project (A link to the areaDetector github page can be found in the footer of this site). The plugin integrated the opencv and zbar libraries for image processing and barcode detection respectively, and wrote detected barcode messages into Process Variables, or PVs. Database integration is the next piece of the puzzle, and current work is focused on adding built in support for MySQL and MongoDB databases, with more specialized solutions being added later. The flexibility of this solution, however, lies in the fact that barcode/QR code information is stored in PVs, which can be accessed through a variety of methods including through a shell script or a python program, meaning that regardless of the database implementation in use at a particular beamline, even if the plugin itself doesn't natively support it, intergating it should not be difficult.


Rowers Meet Dating Website

For my principles of database systems course at Stony Brook University, I worked as part of a small team to create a dating website using HTML, CSS, Javascript, and Bootstrap for the front end portion, and Java and MySQL for the website backend. We used .jsp files and servlets to integrate the html with the output of the jdbc based SQL queries, and we hosted the completed website on an Apache server. Source code for the project is available on my Github account.
